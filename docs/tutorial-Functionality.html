Workarounds

The KA builds include the JSHint and BabyHint workarounds.

JSHint

Khan Academy employs JSHint to enforce a certain code style (i.e. semicolons). Using the ignore JSHint directive disabled this functionality: 

// jshint ignore: start

BabyHint

BabyHint is a project made specifically for Khan Academy to "provide more helpful hints". BabyHint is also used to restrict the usage of certain keywords (i.e. document). BabyHint uses regular expressions to "parse" code which leads to it making incorrect assumptions. Ben Burrill originally found a method of completely disabling BabyHint using a certain regular expression: 

(/\/*/);

Loop Protector

To avoid programs from crashing the webpage, Khan Academy injects it's own code into loops that kill the program if it tries to call more than 1000 iterations. This is an understandable concern but more often than not comes into play when it's not needed. The disableLoopProtector function disables this functionality: 

disableLoopProtector();

Modern Syntax

Although most of the major syntax changes introduces ES5 are impossible to use in a program, variable declarations are an exception. For the most part, however, Khan treats var, let and const as the same. 

// jshint ignore: start
let a = 0;
const b = 1;

Global Objects

JavaScript includes standard built-in objects some of which are not accessible in the default Khan environment.

Browser Console

Documentation

A more powerful alternative to the println function. The console object provides access to the browser's debugging console. The browser console can be opened with keyboard shortcuts:

Windows macOS
Chrome Ctrl + Shift + J ⌘ + ⌥ + J
Firefox Ctrl + Shift + K ⌘ + ⌥ + K
Safari ⌘ + ⌥ + C
Internet Explorer F12

Microsoft Edge and Opera are both chromium-based browsers, refer to Chrome. Linux users should already know how to open the browser console.

Example

Given an object obj:

const obj = {
    a: 1,
    b: 2
}

The traditional method of debugging in the Khan Academy environment is by calling the println function: 

println(obj);

The output of console.log is more useful, especially with more complex objects. 

console.log(obj);


console.table(obj);

Google Chrome and Firefox both offer their own suite of developer tools; Chrome offers Chrome DevTools, and Firefox has its Developer Tools.

Libraries

The following libraries are dependencies for the Khan Academy website, there are no additional wait times or boilerplate code to use them.

The only caveat being that Khan Academy doesn't use the latest version of these libraries. Some "newer" features may be missing.

Banned Properties

Using the BabyHint workaround allows direct access to the following APIs:

  • createElement
  • document
  • location
  • ownerDocument

Storage

The Essentials library does not include a wrapper for the Storage API as it's straightforward. Both Local and Session storage work inside the Khan Academy environment although External Essentials includes the isLocalStorageEnabled and isSessionStorageEnabled functions to conduct sanity checks.
